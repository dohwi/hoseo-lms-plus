# GitHub Flow

- 기본 브랜치에서 최신 내용을 받은 뒤 작업별 브랜치를 만든다.
- 브랜치에서 작업하고, 로컬 확인이 끝나면 커밋한다.
- 커밋 메시지는 `feat: 기능 설명`, `fix: 수정 설명`처럼 접두사는 영어로 쓰고, 내용은 한글로 작성한다.
- 권장 접두사: `feat`, `fix`, `refactor`, `docs`, `test`, `chore`, `style`.
- 작업이 끝나면 브랜치를 푸시하고 Pull Request를 만들어 리뷰 후 머지한다.
- main에는 직접 커밋하지 않고, 작은 단위로 자주 커밋한다.

# 커밋 예시

- `feat: 대시보드 주차 이동 UI 추가`
- `fix: 수강 완료 영상이 미수강으로 표시되는 문제 수정`
- `refactor: 출석 데이터 파싱 로직 분리`
